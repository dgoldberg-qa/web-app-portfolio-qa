# SauceDemo web application QA testing

## About

This repository contains my QA portfolio project based on testing the SauceDemo web application.

The project demonstrates  the complete QA testing process, from planning and test documentation to bug reporting and the final test summary.

## Testing activities

The following QA activities were performed:

- Checklist creation
- Test case design
- Exploratory testing
- Bug reporting
- Test summary report

## Test environment

- **Application:** SauceDemo (https://www.saucedemo.com/)
- **Platform:** PC
- **Operating System:** Windows 11

## Repository structure

- **Checklist** - Functional and exploratory testing checklist
- **Test_cases** - Test cases covering core app features
- **Bug_reports** - Reproducible defects identified during testing
- **Test_summary** - Final testing summary and conclusions

## Project summary

During testing:

- 64 checklist items were executed.
- 5 test cases were created.
- 5 reproducible bugs were reported, including functional, authentication and UI issues.

The testing focused on core website functions, user interface, session management and overall user experience.
Additional testing was performed using the application's special test accounts, which expose different application behaviors for testing purposes.

## Objective

The goal of this project was to practice manual QA testing by documenting the testing process and providing structured feedback for a test web application.
