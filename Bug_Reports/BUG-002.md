# Protected action can be performed after session expiration

**Severity:** Major

**Priority:** High

**Environment:** Brave v1.92.140

**Device:** PC

**OS:** Windows 11

**Preconditions:**

User is logged in as standard_user.
The session has expired due to long inactivity.

**Steps to reproduce:**

1. Leave the website open for a long period of time.
2. Click Add to cart on any product.
3. Observe the application behavior.
4. Log in again.
5. Open the shopping cart.

**Expected result:**

The application should redirect the user to the Login page before executing any protected action. The product should not be added to the cart.

**Actual result:**

The product is added to the cart, but the user is redirected to the Login page only after the action is performed.

**Frequency:**

Always.
